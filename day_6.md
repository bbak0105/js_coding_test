### 프로그램의 성능 측정 방법

<br/>

1. 시간 복잡도: 연산 횟수를 측정
> for문시에 for문 돌아간 만큼 BIG-O 표기법대로 된다.
O(n)의 시간 복잡도 처리

> O(n2): 이중 포문

> 일반적으로 횟수가 10억을 넘어가면 1초 이상의 시간이 소요된다.
* O(n): 약 1,000번의 연산
* O(nlogn): 약 10,000번의 연산
* O(n2): 약 1,000,000번의 연산
* O(n3): 약 1,000,000,000번의 연산

> BIG-O 표기법으로 시간 복잡도를 표기할 때는 가장 큰 항만을 표시하며, 가장 큰 항에 붙어있는 계수는 제거한다.
* O(3n제곱 + n) = O(n제곱)

<br/>

2. 공간 복잡도: 메모리의 양을 측정

> 코딩 테스트에서 메모리의 크기를 나타낼 때는 일반적으로 MB 단위로 표기한다.

* int a[1000] : 4KB
* int a[1000000] : 4MB
* int a[2000][2000]: 16MB
